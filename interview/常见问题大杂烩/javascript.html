Javascript ​

JavaScript 是一门跨平台、面向对象的动态的弱类型的轻量级解释型语言，是一种基于对象和事件驱动并具有相对安全性的客户端脚本语言。

ajax ​

ajax是异步的javascript和xml (async JavaScript and xml)

使用javascript通过异步的方式进行前后台数据交互，相比较于传统的将数据直接包含在页面中的方式，拥有更好的交互体验，可以在不刷新不跳转页面的前提下实现页面数据的更新。

  1. ajax是现在进行项目开发最主要的手段，在之前项目中的话，jquery中使用$ajax发送请求，在vue中使用axios发送请求，在微信小程序中也会用到ajax
  2. 一般发送的话主要有get、post、delete、put等方法，返回的结果一般是异步的，但是我们一般通过async await的方式去接收，可以更好地以一个同步的方式去写一个东西。

与传统form表单请求的区别

  • form表单：提交服务器必须等待服务器响应成功之后才可以继续操作。——同步
  • ajax：在响应服务器的过程中，用户可以继续操作页面。——异步

var 和let的区别 ​

var是es6之前用来声明变量的关键字，let是es6中新增的用于声明变量的关键字，解决了var存在的一些问题

  1. var有变量提升(在每一个代码块中，用var声明变量和用function声明的函数会被优先解析)，let没有；
  2. var可以重复声明，let不可以；
  3. var声明的全局变量会作为window对象的属性，let不会；
  4. let可以识别块级作用域，var不会，({}会识别花括号)

箭头函数 ​

是在es6中新增对于匿名函数的一种简写方式，可以把形参放在箭头的左边，返回值或者代码块放在箭头的右边

箭头函数没有自己的this指向和argument对象，指向的是上下文对象

this ​

this是js中除了箭头函数以外，每个函数内部默认可以访问的一个变量，表示的是当前函数的调用者，所以在函数未被调用的时候this是没有意义的。

  • 普通的函数调用：this指的是window，
  • 对象的方法调用，this指的是当前的对象，
  • 实例化的方式(new):this指的是最终实例化的对象
  • call/apply在函数调用时指定this，
  • bind是在函数声明时去指定this

call和apply的区别 ​

如果函数还需要传递其他的参数，call正常依次传递，apply需要将参数放置到一个数组当中

localStorage和sessionStroage区别 ​

localStorage和sessionStroage都是h5中定义的特性，用于方便进行数据的缓冲，尤其是开发移动端应用

相比cookie可以由服务器去设置和获取，进行用户鉴权操作，localStorage和sessionStroage只能由js操作，localStorage和sessionStroage存储的数据量也会更大。

  • 都是用来本地存储，同源<协议/域名/端口都相同>
  • 存储方式不同
    • cookie携带在域名后面
    • localStorage和sessionStorage直接存储在浏览器
  • 存储数据大小
    • cookie只能保存4kb
    • localStorage和sessionStorage可以保存5M
  • 数据有效期不同
    • cookie超过设置的时间才会过期
    • sessionStorage只要浏览器关掉就消失，保存一次会话的数据
    • localStorage永久保留，除非手动移除

localStorage 、sessionStorage 、cookie共同点，每个浏览器独立保存，每个域名下也是独立保存的，不同浏览器、域名之间都是互相不影响的，也无法使用

回流和重绘 ​

回流:也叫重排，在dom渲染过程当中，计算每个盒子的宽高位置进行渲染的一个排布

重绘:将具体的虚拟dom渲染到真正的屏幕中，也就是分配每个像素点的颜色

回流必定引起重绘，重绘不一定会引起回流

什么是原型 ​

在js中，每个对象都有原型，原型本身也是一个对象，object.prototype是所有原型链的最终指向

访问对象的属性和方法的时候如果当前对象身上没有，就会沿着它的原型链一直向上查找

在js的内置对象，比如数组，js会将所有的方法添加到数组对象的原型(Array.prototype)上每个数组就都可以使用这些方法了

发送请求的客户端(浏览器端)地址和接收响应的服务器端地址，如果协议、地址、端口号三者有任意一个不一致则会触发跨域问题，违反同源策略

  1. jsonp，不再借助于ajax，而是使用script来完成请求的发送
  2. 在服务器中直接设置允许跨域，一般需要设置一系列头信息，最重要的是Accepct-Control-Allow-Origin:*
  3. 使用请求代理，有前端自己的服务器(自己开发的一般为localhost:8080)去转发请求，vue项目:vue.config.js中，uniapp中:manifest.json中进行配置

promise ​

promise是用来解决回调地狱问题，因为js单线程异步机制的原因，我们要接收异步操作的结果一般只能是在回调函数中获取，如果要基于这个结果继续发送异步请求，就会造成代码层层嵌套，这就是回调地狱。

promise通过将异步请求封装，在获取异步操作结果的时候通过then方法获取，并且在then的回调函数中可以直接返回一个值，返回的值会被继续封装为promise对象，所以可以继续以.then的方式调用，从而脱离回调地狱层层嵌套的逻辑

当然在具体的使用过程中我们可以借助async await，像填写同步代码一样处理异步的逻辑。

闭包 ​

让函数内部的局部变量在函数调用完成之后不被销毁的一种方式，可以通过在函数内部再定义一个函数，通过内部函数引用外部函数的变量，只要内部函数不被销毁，外部函数的变量就不会被销毁

js
function fn(){
	var num=10;
	return function(){
	console.log(num++)
	}
}
fn();//值是一个函数
fn()();//打印一个10

MVVM ​

设计思想，vue是参考这个设计思想开发出来的框架，模型——视图——视图模型

  1. 将模型（Model）转化成视图 (View)，即将后端传递的数据转化成所看到的页面。实现的方式是：数据绑定。
  2. 将视图 (View)转化成模型(Model)，即将所看到的页面转化成后端的数据。实现的方式是：DOM 事件 监听。这两个方向都实现的，我们称之为数据的双向绑定。

目的：

项目复杂度越来越高之后，MVC中的Controller会变得越来越臃肿，难以维护，所以将其中的数据和逻辑处理抽离出的MVVM更佳

同步、异步 ​

  1. js中的代码分为同步代码和异步代码，在代码最终执行的时候先执行同步代码，再执行异步代码(js中常见的异步操作:setIntaval、setTimeout、事件、ajax的send请求发送和对成功加载事件的监听)
  2. js的运行机制:单线程异步机制，所谓的单线程就是同一个时刻只执行一件事情，所以对于一些开销比较大的操作，js会以异步的方式来执行，具体当js执行器解析将要执行的代码的时候，会将同步代码放置到执行栈中依次执行，然后将异步的任务放到等待区域，当执行栈清空之后就会判断有没有可以执行的异步任务，如果有就将异步任务添加到任务队列当中，添加完成之后，如果任务队列中有可以执行的任务就将任务队列中的任务放入执行栈当中依次执行，不断循环整个过程，执行完之后继续判断有没有执行的异步代码。不断循环的过程叫做事件循环。
  3. promise本身内部是同步的
  4. 所有的代码都分为宏任务微任务，宏任务包括整体代码块的执行，setTimeout、setInterval、setImmidiate、I/O、微任务包括promise的then和process.nextTick、执行完一个宏任务之后就要读取当前可以执行的微任务，微任务清空之后继续读取宏任务队列

如何删除一个对象 ​

js为每一个引用类型的数据设置引用计数的统计，js的垃圾回收机制会在对象的引用计数变为0的时候将它的内存空间释放掉，所以要想删除一个对象就需要将将它的引用计数降为0就可以了，所以一般选择将占位符赋值给变量 null

什么是防抖和节流，如何实现 ​

在js中，某些事件的触发可能会频率过高，比如mousemove、scroll，但是我们可能并不需要这么高的触发频率，就可以通过防抖和节流两种方式，来处理这种问题。

防抖:让事件以更低的频率去执行，在事件触发之后，设置一个开关，延迟一段时间之后再打开开关，这样的话就不会频繁的触发了

节流:在整个事件触发的周期内，在事件触发之后，设置一个开关，让事件只触发一次或者两次，只要事件还在触发就会不停的去设置关闭开关，最后在延迟一段时间后再打开开关
